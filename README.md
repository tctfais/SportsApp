# SportsApp
<H1>Описание</H1>
Приложение для большой транснациональной компании, производящей спортивные товары (одежда, обувь) и инвентарь. Для популяризации и продвижения своей продукции менеджмент решает, что необходимо освоить новые каналы для информирования покупателей о выходе новых товаров и стимулировать спрос на новые вещи. В компании уже разработаны приложения для покупки товаров, а также узкоспециализированные приложения для некоторых видов спорта. 


<H1>Основные требования</H1>
<ol>
    <li>Реализация социальных компонентов в приложении. Оно должно формировать социальные группы по интересам, которые будут самоподдерживаться и в которых участники будут общаться и влиять друг на друга. Одна из целей — формирование такого образа бренда в глазах участников, который позволит нашим товарам оставаться в фаворитах при выборе из прочих равных. 
    <li>Предоставление информации о характеристиках тренировки, сравнение с прошлыми тренировками, сравнение с людьми в регионе, с профессиональными спортсменами. Основное сравнение — с самим собой для стимулирования результатов</li>
    <li>Возможность поиска людей по схожим интересам, поиск людей, которые тренируются в том же месте, имеют те же маршруты или которые тренируются прямо сейчас, для формирования групп для совместных занятий</li>
    <li>Возможность указания своего спортивного инвентаря (обувь, снаряды) для подсказок по составлению тренировок или своевременного обновления обуви</li>
    <li>Формирование и подсказки по составлению тренировок и их расписания</li>
    <li>Уведомление друзей о ваших новых успехах</li>
    <li>Геймификация</li>
    <li>Внедрение промоакций и новостей спорта в зависимости от характера тренировок для вовлечённости</li>
    <li>Возможность подключения сторонних устройств для отслеживания тренировок (датчик сердцебиения, кислорода и так далее)</li>
    <li>Лёгкая интеграция существующих приложений компании для облегчения продаж</li>
    <li>Возможность вставки региональных промоакций</li>
    <li>Возможность проведение соревнований с большим количеством участвующих</li>
    <li>Подключение дополнительных устройств, которые помогают следить за состоянием организма, для продвинутых спортсменов</li>
    <li>Интеграция с фитнес-функциями телефона</li>
</ol>


<H1>Бизнес цели</H1>
Компания стремится предоставить каждому спортсмену — от профессиональных спортсменов и любителей бега и йоги до детей на детской площадке — возможность, продукцию и вдохновение для достижения спортивных целей, на которые каждый способен. У любого человека есть потенциал для великих свершений. Если у Вас есть тело - значит Вы спорстмен. Исходя из этого можно сформировать основые бизнес-цели:
<ol>
    <li>Популяризация и стимулирование регулярного занятим спортом. Приложение должно вызывать интерес к спорту различными способами: достижения друзей, рекорды других участников, напоминамия о том что человек давно не тренировался, интересные статью, новости спорта</li>
    <li>Создание спортивной экосистемы. Все модули и компоненты системы должны быть между собой связаны. Каждое приложение должно иметь плавный переход к другому. Пользотельский путь должнен быть максимально короткий между всеми модулями.</li>
    <li>Повышения удобства подбора спортивного инвентаря. В системе продажи товаров должны быть учтены все возможные характертеристики товаров. Должны быть преднастроенные профили которые позволят человеку получить относительно его биометрических параметров готовые комплеты для тренировок.</li>
    <li>Освоение новых каналов и способов информирования о новых товарах. Интеграция с социальными сетями, push-рассылки, e-mail рассылки. Должна быть внедрена концепция "daily app", которая позволит сделать продвижение товара для пользователя более незаметной. Возможно внедрение программы утилизации старого спортивного инвентаря для получения скидки на новый.</li>
</ol>


<H1>Функциональные требования</H1>
<ol>
    <li>Мультиязычность: приложение должно быть доступно на нескольких языках, чтобы удовлетворить потребности глобальной аудитории</li>
    <li>Автоматическое отслеживание и анализ тренировок: приложение должно автоматически отслеживать и анализировать данные о тренировках, например, расстояние, скорость, время, пульс, количество шагов и т.д. Для этого могут использоваться датчики на мобильных устройствах или сторонние устройства</li>
    <li>Рекомендации по тренировкам и питанию: на основе данных о тренировках и питании, приложение может предоставлять рекомендации по улучшению результатов и достижению спортивных целей</li>
    <li>Уведомления и напоминания: приложение может отправлять уведомления и напоминания о предстоящих тренировках, целях и достижениях, а также о необходимости обновления спортивного инвентаря</li>

    <li>Ограниченность политической воли руководителей проекта для принятия волевых решений</li>
</ol>




<H1>Используемые технологии для разработки</H1>

**Типовой микросервис для унификации процесса разработки должен состоять из следующих компонентов:**

<ol>
    <li>Компонент для хранения данных: это может быть реляционная или NoSQL база данных</li>
    <li>API-компонент: это компонент, который будет обрабатывать запросы клиентов и взаимодействовать с базой данных для получения или изменения информации. Для написания API можно использовать любой язык программирования, который поддерживает создание RESTful API</li>
    <li>Аутентификационный компонент: это компонент, который будет обеспечивать безопасность системы и аутентификацию пользователей. Для этого можно использовать существующие библиотеки для аутентификации, такие как OAuth или JWT</li>
    <li>Компонент для отправки уведомлений: это компонент, который будет отправлять уведомления пользователям о состоянии их заявок. Для этого можно использовать существующие сервисы отправки уведомлений</li>
    <li>Компонент для мониторинга и логирования: это компонент, который будет отслеживать работу системы и записывать логи. Для этого можно использовать существующие инструменты для мониторинга и логирования</li>
</ol>
    
**Перечень допустимых на проекте технологий:**

<li>Библиотеки для аутентификации: OAuth, JWT</li>
<li>Сервисы отправки уведомлений: Twilio, SendGrid</li>
